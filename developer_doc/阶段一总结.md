# 阶段一总结：基础链路验证

## 一、目标回顾

本阶段对应 `任务执行顺序.md` 中的阶段一，包含以下任务：
- **T-007**：在 `MainActivity` 中添加跳转到 `WebViewActivity` 的按钮。
- **T-008**：准备测试 HTML 页面（本地 assets 或远端），用于验证 WebView 和 JS Bridge。
- **运行测试**：验证 `WebViewActivity`、`WebViewBridge`、`WebViewSecurity` 三文件，确认 WebView 能加载页面且 JS Bridge 能弹出 Toast。

## 二、完成的工作

### 1. 创建 WebView 模块文件
- `WebViewActivity.kt`：加载本地测试页面（`file:///android_asset/test.html`），配置 WebView 安全策略，添加 JS 接口。
- `WebViewBridge.kt`：提供 `@JavascriptInterface` 方法（`showToast`、`startAutoUpload`、`stopAutoUpload` 等）。
- `WebViewSecurity.kt`：扩展函数 `applySecuritySettings()`，设置域名白名单、禁用混合内容、移除危险接口。

### 2. 修改 `MainActivity`
- 添加 Compose 按钮，点击启动 `WebViewActivity`。

### 3. 准备测试页面
- 在 `app/src/main/assets/` 下创建 `test.html`，包含调用 JS Bridge 的按钮。

### 4. 解决模拟器 WebView 渲染进程崩溃
- 在 `AndroidManifest.xml` 中为 `WebViewActivity` 禁用硬件加速（`android:hardwareAccelerated="false"`）。
- 设置主题为 `Theme.AppCompat.Light.NoActionBar`。
- 在 `WebViewActivity` 中添加 `onRenderProcessGone` 回调，优雅处理崩溃。

### 5. 配置网络转发
- 使用 `adb reverse tcp:8080 tcp:8080`，让模拟器通过 `localhost:8080` 访问宿主机 `live-server`。

## 三、遇到的问题及解决方案

| 问题 | 解决方案 |
|------|----------|
| `ActivityNotFoundException` | 在 `AndroidManifest.xml` 中注册 `WebViewActivity` |
| `IllegalStateException: Theme.AppCompat required` | 为 `WebViewActivity` 单独指定 `android:theme="@style/Theme.AppCompat.Light.NoActionBar"` |
| WebView 加载本地 assets 页面时渲染进程崩溃 | 禁用硬件加速 `android:hardwareAccelerated="false"`，并设置软件渲染 |
| `net::ERR_NAME_NOT_RESOLVED` | 使用 `adb reverse` 转发端口，避免模拟器 DNS 问题 |
| 页面空白但模拟器浏览器能访问 | 简化测试 HTML 内容，逐步排查；最终发现是硬件加速导致，禁用后正常 |
