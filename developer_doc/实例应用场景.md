# 实例应用场景

> 本文档重新归纳了数据采集与处理的核心流程，替代原始 6 点描述，并补充决策逻辑与异常处理。

## 一、核心流程模块

| 数据源 | 是否经过功能处理 | 存储目标选项 | 备注 |
|--------|----------------|--------------|------|
| 手机存储文件 | 否 | 本地数据库 / 远程服务器 | 纯采集存储，不转换 |
| 手机存储文件 | 是 | 本地数据库 / 远程服务器 | 先执行处理器链，再存储结果 |
| 传感器实时数据 | 否 | 本地数据库 / 远程服务器 | 原始数据入库或直传云端 |
| 传感器实时数据 | 是 | 本地数据库 / 远程服务器 | 经本地处理（如平滑、阈值检测）后存储 |
| 远程服务器已有数据 | 是 | 本地数据库 / 远程服务器 | 下载后处理，结果可存本地或回写云端 |
| 本地数据库已有数据 | 是 | 本地数据库 / 远程服务器 | 读取已存数据，处理后更新本地或同步云端 |

## 二、详细描述（替代原 6 点）

### 1. 采集原始数据并直接存储
- 从**手机存储文件**或**传感器**获取原始数据。
- 根据配置或用户选择，将数据存入**本地数据库**和/或**远程服务器**（支持仅本地、仅远程、双写）。
- 若网络不可用，自动降级为仅本地存储，并加入同步队列。

### 2. 采集原始数据后经功能处理再存储
- 从文件或传感器获取原始数据后，送入**数据处理流水线**（一个或多个顺序执行的处理器，如滤波、特征提取、分类）。
- 处理成功则得到**转化数据**或**消费数据**，根据配置存入本地数据库和/或远程服务器。
- 处理失败时可选择：跳过存储、重试处理、或降级为存储原始数据。

### 3. 从已有数据源获取数据后处理再存储
- 数据源可以是**本地数据库**或**远程服务器**（需支持按条件查询获取）。
- 将获取的数据输入处理流水线（与上条相同）。
- 处理结果可写回**原数据源**或**另一个存储位置**（例如从服务器下载聚合数据，处理后存入本地用于离线展示）。

### 4. 统一决策与约束
- 所有“选择”行为由**策略模块**决定：可基于数据类型、数据大小、用户偏好、网络状态、电量等动态调整。
- 支持**运行时修改策略**（例如通过 WebView 前端下发配置）。
- 所有存储操作均记录**输入-输出映射**，便于溯源与缓存。

### 5. 异常与离线处理
- 当目标为远程服务器但无网络时：本地持久化请求，由 `WorkManager` 在恢复网络后自动同步。
- 当处理过程抛出异常：记录错误，可选择重试（指数退避）或标记为“待人工处理”。

## 三、与现有架构的对应关系

| 抽象概念 | 具体实现 |
|----------|----------|
| 手机存储文件 | `RawData` 子类（如 `RawImageFile`），通过 `FileUtils` 读取 |
| 传感器数据 | `RawHeartRate`、`RawAccelerometer` 等，由 `SensorCollector` 采集 |
| 功能处理 | `DataProcessor` 接口实现类，通过 `FlexiblePipeline` 串联 |
| 本地数据库 | Room 三张表：`raw_data`、`transformed_data`、`consumable_data` |
| 远程服务器 | Netlify API：`/api/upload`、`/api/sensor-data`、`/api/mappings` 等 |
| 选择逻辑 | `ProcessingStrategy` 类，可在 `WebViewBridge` 中动态修改策略参数 |

## 四、总结（需求文档用语）

> **数据采集与处理系统需支持以下操作模式：**
> - 从手机存储文件或传感器获取原始数据，可选是否经处理流水线，最终将结果存入本地数据库和/或远程服务器。
> - 从本地数据库或远程服务器读取已有数据，经处理后写回任意存储目标。
> - 所有存储决策（目标位置、是否处理）由可配置的策略模块决定，并支持离线缓存、失败重试与输入输出映射记录。
