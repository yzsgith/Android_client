# 阶段二总结：传感器模拟采集与存储

## 一、目标回顾

本阶段对应 `任务执行顺序.md` 中的阶段二，包含以下任务：
- **T-013**：添加 `AndroidManifest.xml` 权限（`INTERNET`、`BODY_SENSORS`）。
- **T-009 / T-010**：完善 `HeartRateCollector` 和 `AccelerometerCollector`，实现模拟数据生成。
- **T-011**：确认 `SensorDataRepository` 能正确将数据存储到 Room 的 `raw_data` 表。

## 二、完成的工作

### 1. 添加权限
在 `AndroidManifest.xml` 中添加：
```xml
<uses-permission android:name="android.permission.INTERNET" />
<uses-permission android:name="android.permission.BODY_SENSORS" />
```
### 2. 实现采集器（含模拟数据）
HeartRateCollector.kt
优先使用真实心率传感器（Sensor.TYPE_HEART_RATE）。

若无传感器，启动协程模拟数据：每秒生成 60~100 之间的随机心率，通过 Flow<RawHeartRate> 输出。

AccelerometerCollector.kt
优先使用真实加速度计（Sensor.TYPE_ACCELEROMETER）。

若无传感器，模拟数据：每 200ms 生成范围 -2~2 的随机加速度值（x, y, z），通过 Flow<RawAccelerometer> 输出。

### 3. 数据存储
SensorDataRepository 提供 insertRawData(raw: RawData) 方法，内部调用 RawDataEntity.fromDomain(raw) 转换为实体，并通过 RawDataDao 插入 raw_data 表。

在 MainActivity 中启动两个采集器，分别收集其 Flow，调用 repository.insertRawData()。

### 4. 验证
运行 App，Logcat 过滤 Sensor 标签，观察到心率（每秒）和加速度（每200ms）的保存日志。

使用 App Inspection 查看 app_database 中的 raw_data 表，确认数据已写入，type 字段正确区分 heart_rate 和 accelerometer，dataJson 包含正确的 JSON 内容。

## 三、遇到的问题及解决方案
问题	解决方案
模拟器无真实心率传感器	在 HeartRateCollector 中检测 getDefaultSensor 返回 null 时自动启动模拟数据生成协程
加速度模拟数据过于规律	使用 Random 生成随机值，并设置 delay(200) 模拟真实采集频率
数据未写入数据库	确保 insertRawData 使用 withContext(Dispatchers.IO) 在 IO 线程执行；检查 RawDataEntity.fromDomain 是否正确序列化为 JSON
Logcat 日志混杂	使用组合过滤 tag:Sensor -Accelerometer 单独查看心率日志
