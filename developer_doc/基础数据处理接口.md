# 基础数据处理接口

> 本文档列出完成所有功能所需的核心数据处理接口，包括单步处理器、流式处理器、处理器链、策略接口及注册中心。  
> 所有接口文件均需添加 `// TODO` 注释，描述其职责与待实现内容。

## 二、接口详细说明

### 1. DataProcessor<IN, OUT>
- **职责**：将输入数据 `IN` 转换为输出数据 `OUT`，不关心转换的复杂程度。
- **典型实现**：`HeartRateSmoother`（原始心率 → 平滑心率）、`ImageCropper`（原始图像 → 裁剪图像）。
- **协程支持**：使用 `suspend` 方法，便于执行耗时操作（如文件 I/O、模型推理）。

### 2. StreamingProcessor<IN, OUT>
- **职责**：处理连续数据流（如每秒采集的传感器数据），返回转换后的流。
- **典型实现**：`RealTimeStepCounter`（实时加速度流 → 步数流）、`HeartRateMonitor`（心率流 → 告警流）。

### 3. ProcessorChain
- **职责**：将多个 `DataProcessor` 串联执行，上一个的输出作为下一个的输入。
- **使用场景**：原始数据需要经过滤波、特征提取、分类等多个步骤时。
- **注意**：链中处理器类型需匹配（`OUT` 与下一个的 `IN` 一致），否则运行时可能抛异常。

### 4. ProcessingStrategy
- **职责**：根据数据类型、用户设置、网络状态、电量等动态决策：
    - 是否进行本地处理（否则直接存储原始数据或上传云端）。
    - 处理后的结果存储目标（仅本地、仅远程、双写）。
- **实现示例**：`RuleBasedStrategy`（基于规则表）、`AdaptiveStrategy`（基于机器学习）。

### 5. ProcessorRegistry
- **职责**：维护一个从 `RawData` 类型到 `ProcessorChain` 的映射，便于在采集流程中自动查找并执行对应处理链。
- **扩展性**：支持运行时动态注册新的处理器链（例如从服务器下载配置）。

## 三、使用示例（伪代码）

```kotlin
// 1. 定义处理器
val smoother = HeartRateSmoother()
val analyzer = HeartRateAnalyzer()

// 2. 构建链
val chain = ProcessorChain()
    .add(smoother)   // RawHeartRate → SmoothedHeartRate
    .add(analyzer)   // SmoothedHeartRate → HeartRateAlert

// 3. 注册
ProcessorRegistry.register(RawHeartRate::class, chain)

// 4. 执行
val raw = RawHeartRate(115, 0.95f, System.currentTimeMillis())
val alert = ProcessorRegistry.getChain(RawHeartRate::class)?.execute(raw) as HeartRateAlert
